# Universal cross-domain principles

Forty recurring patterns observed across many fields (physics, biology, computer science, economics, ecology, …). Used in **Stage 3 (Expand)** to suggest less-obvious candidate domains: pick the principles that match your problem, then explore the domains where each principle appears.

Format per entry: principle name, what it does, domains where it shows up, keywords to grep for.

## How to use this table in Stage 3 (Expand)

1. Reread the problem's `analogical_hooks`, `behavior_chain`, and SAPPhIRE `effect`.
2. Scan the principle list. For each principle that plausibly describes the problem's underlying pattern, jot the principle # and the matching keyword.
3. For each matched principle, look at the **other domains** listed — these are your candidate domains.
4. Prefer principles where the matched domains are **furthest** from the home domain. If the problem is in software engineering and "Caching" matches, biology and economics are far more interesting candidates than the same field.
5. A single problem usually matches 3–5 principles. Use that to populate candidate domains across multiple lenses.

## How to use this in Stage 4 (Mine)

When mining a candidate domain, search the table for principles tagged with that domain. The principle gives you a starting hypothesis for *what mechanism to look at* — but you still need to find a specific named mechanism (e.g., not "feedback in biology" but "baroreceptor reflex").
